## 👤 我是谁
🧩 独立开发者，专注把“想法 → 可执行计划 → 可交付代码 → 上线增长”做成可复用系统：**Skills（技能）+ Agents（代理）+ Workflows（工作流）+ Hooks/Rules（质量门控）**。

## ⚡ 我在做什么
- 🔎 把公开信号转成可落地机会：PRD / 执行清单 / 风险评估 / 增长策略
- 🧠 搭建“一人公司”AI 工作流系统：规划 → 开发（TDD）→ 审查 → 发布
- 🔁 让同一套方法论在多种 AI 编程运行时复用（Claude Code / Cursor 等）

## 🧱 代表项目（Real Projects）

### 🚀 SuperOPC — 一人公司超级操作系统
- 🔗 仓库：https://github.com/Gjts/SuperOPC
- 🧠 一句话：面向独立创始人的 AI Operating System，用 **技能系统 + 代理编排 + 斜杠命令 + Hooks/Rules 质量门控**，把产品研发与商业运营流程“系统化、可复用、可持续迭代”。
- ✨ 真实亮点（来自仓库现状）：
  - 🧰 **51 个技能**（产品开发 / 工程质量 / 商业运营 / 市场情报 / 学习进化）
  - 🤖 **15 个专业代理**（规划、执行、审查、调试、安全审计、UI 审计、文档等）
  - ⌨️ **23 个斜杠命令**（`/opc-start` / `/opc-plan` / `/opc-build` / `/opc-ship` …）
  - 🧪 **多运行时导出**：Claude Code 原生 + 导出到 Cursor/Windsurf/Copilot/Codex 等（脚本一键转换）
- 🧭 推荐入口：先看 `README` 的“快速开始”，再看 `ROADMAP.md`

### 🧿 SuperOPC Marketplace — 插件上架/索引仓库

### 📚 AI Daily Opportunities — 每周 10 个可验证的 2C AI 机会（带完整落地文档）
